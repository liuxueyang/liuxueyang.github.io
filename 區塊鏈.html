1.2 帳號管理

1.2.1 把帳號從 MetaMask 中導入到 Geth

  • https://github.com/ethereum/go-ethereum/wiki/Managing-your-accounts
  1. 導出 Private Key

    Private Key 保存到一個文件裏面： liu.csv 。

  2. 導入
    geth account import liu.csv

    設置一個密碼，我選擇不相信自己的記憶，老老實實把密碼記在了 KeePass 裏面。

    當然，如果帳號是在 Rinkeby 上面的話「我的這個帳號就是 Rinkeby 上的」，應該這樣做： 

    geth --rinkeby account import liu.csv

1.2.2 永久 unlock 賬戶

  • https://ethereum.stackexchange.com/questions/11102/unlock-account-permanently
  • https://github.com/ethereum/go-ethereum/wiki/Managing-your-accounts
personal.unlockAccount("address", "pass", 0);

只在當前的 geth console 中有效，重啓 geth 之後解鎖狀態失效。

1.3 設置 Geth 的 coinbase

假設我想要設置的地址是 eth.accounts[2] ： 

miner.setEtherbase(eth.accounts[2])
  • https://ethereum.stackexchange.com/questions/12181/how-do-you-configure-geth-to-use-coinbase

不過這樣設置好像不是永久的，重啓 Geth 的時候 coinbase 會變成原來的。其實啓動 Geth 的時候可以用下標的形式指定 eth.accounts 中的某個地址：「指定下標爲 2 的地址作爲 coinbase 」 

geth --rinkeby --etherbase 2 console 2>> rinkebyEth.log

3 Troubleshooting

3.1 Invalid Address Error when interacting with smart contract

原因是沒有設置 web3 的默認賬戶。解鎖一個賬戶，然後把它設置成默認賬戶即可。「我已經把 coinbase 賬戶永久解鎖了」 

web3.eth.defaultAccount = eth.coinbase

3.2 啓動 Ethereum Wallet

這個軟件不是一般的難用，畢竟不是原生的，可能會出現各種奇怪的問題。比如啓動的時候一直卡住進不去主頁面什麼的。開始覺得可能是版本太低，然後升級到了 0.11.1，還是不行。又降級到了 0.11.0，直接啓動還是不行，最後這樣啓動纔可以： 

cd /Applications/Ethereum\ Wallet.app/Contents/MacOS/
./Ethereum\ Wallet --rpc ~/Library/Ethereum/rinkeby/geth.ipc

行吧，至少可以使用了。其實日誌裏面還是會有奇怪的錯誤，比如： 

[2018-08-01T16:38:39.846] [ERROR] ClientBinaryManager - Sanity check failed for Geth Error: Unable to find "1.8.11" in
Geth output

或者： 

(node:14680) MaxListenersExceededWarning: Possible EventEmitter memory leak detected. 501 message listeners added. Use
emitter.setMaxListeners() to increase limit

行吧，至少能打開了。:( 就這個軟件，折騰了半天。。也還是有一點兒成就感吧。。（大概。
