2 Net Techniques

在使用 Common Lisp 自底向上地編程的時候，實際上是在擴展 Lisp 語言本身，底層的程序爲上層的應用提供了一門幾乎嶄新的語言，這使得最終的程序更具有擴展性，畢竟，有什麼比能夠完全滿足個性化需求的程序設計語言更有復用價值呢？

3 A New Approach

與其依賴提前做好的計劃，不如編寫更加抽象的程序，使得它能夠輕鬆地應對不斷變化的需求，把錯誤的代價降到最低。Common Lisp 正好提供了這種抽象的特性。

程序依賴更少的計劃意味着採用了更好的設計。

  • 如果採用函數式的編程，那麼也就意味着可以把 Bug 控制在一個 更小的範圍內 ；
  • 如果採用更抽象的語言，很多 Bug「例如懸掛指針」其實可以完全避免，剩下的 Bug 其實就更加容易解決；
  • 如果有交互式的編程環境，就可以更方便地解決 Bug，而不用忍受編輯、編譯、測試這種很長的循環流程；

Common Lisp 本身就具有這些特性。

從 Common Lisp 中借鑑的很多特性已經出現在了主流的程序設計語言中，比如：垃圾回收、交互式環境、run-time typing 等等。
